# Livestream Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the `.feed-placeholder` static content in `public/index.html` with a live MoQ video subscriber for broadcast `friend.fish/tank` on `relay.cloudflare.mediaoverquic.com`, and add the `moq-cli` publisher binary to the Nix dev shell.

**Architecture:** Single-file HTML change in `public/index.html`: replace the placeholder's inner DOM with `<moq-watch>` (custom element from `@moq/watch`, loaded as ESM from jsDelivr) wrapping a `<canvas>`, with the existing fish-icon/headline/bubbles markup repurposed as an "offline" overlay. A small `<script type="module">` subscribes to the element's signals (`broadcast.video.media`) and toggles an `.is-live` class on the placeholder. Separately, `flake.nix` grows to consume `moq-dev/moq` as a flake input, exposing `moq-cli` in the dev shell.

**Tech Stack:** vanilla HTML/CSS/JS in a single file; `@moq/watch` via jsDelivr `+esm`; Nix flake with `moq-dev/moq` as an input; Cloudflare Workers static assets (no Worker changes).

**Spec:** `docs/superpowers/specs/2026-05-16-livestream-design.md`

**Testing note:** This project has no automated test infrastructure (verified: no `test/` dir, no test runner in `package.json`). Verification is manual smoke testing per the spec. The spec explicitly excludes adding a test harness for this feature — that decision is locked in. Each task ends with concrete manual checks plus a commit.

## Plan structure

Three tasks, each one commit. Order matters: Task 1 unlocks the publisher locally (needed to verify Tasks 2 & 3 end-to-end), Task 2 reshapes the DOM without behavior change, Task 3 wires up the live behavior.

| # | Task | Files |
|---|---|---|
| 1 | Add `moq-cli` to the Nix dev shell | `flake.nix`, `flake.lock` |
| 2 | Restructure `.feed-placeholder` markup + supporting CSS | `public/index.html` |
| 3 | Wire up `<moq-watch>` subscriber + state machine | `public/index.html` |

After Task 3: page is live. With the publisher running, video appears within seconds; without, the overlay says "connecting…" or "stream offline".

## Task 3: Wire up `<moq-watch>` subscriber + state machine

**Files:**
- Modify: `public/index.html` (add a new `<script type="module">` near the bottom of `<body>`)

**Context:** Task 2 left the page with a `<moq-watch>` element on it but no library loaded — it's an unknown custom element rendering empty. This task adds a `<script type="module">` that:

1. Imports the `@moq/watch` element registration from jsDelivr `+esm`.
2. Subscribes to the element's `broadcast.video.media` signal via the element's `signals.run(...)` effect API.
3. Toggles the `.is-live` class on `#feed-placeholder` when a `MediaStream` is produced.
4. Updates the overlay subtitle between "connecting…" (no frames ever seen) and "stream offline" (was live, now gone).
5. On click while live, requests fullscreen on the canvas.

The script lives in its own `<script type="module">` block, separate from the existing classic `<script>` (which runs from line 715 to ~1253). Module scripts are deferred by default and don't share scope with the classic script, which is exactly what we want — the existing globals (`initData`, `capFeedingLogHeight`, etc.) stay where they are.

The jsDelivr `+esm` endpoint rewrites the package's bare imports (`@moq/lite`, `@moq/hang`, `@moq/signals`) into sibling `+esm` URLs automatically, so the single import below is sufficient — no import map required.

The library version is pinned to `@0.2` (current latest is `0.2.11`); the `+esm` endpoint resolves that to the latest 0.2.x.

```html
  </div>
  <div class="lightbox-img-wrap" onclick="if(event.target===this)closeLightbox()">
    <img id="lightbox-img" src="" alt="photo">
  </div>
</div>

<script type="module">
  import "https://cdn.jsdelivr.net/npm/@moq/watch@0.2/element.js/+esm";

  const el = document.querySelector('moq-watch');
  const box = document.getElementById('feed-placeholder');
  const text = box.querySelector('.feed-overlay-text');
  const canvas = el.querySelector('canvas');

  let everLive = false;

  el.signals.run((effect) => {
    const stream = effect.get(el.broadcast.video.media);
    const live = !!stream;
    if (live) everLive = true;
    box.classList.toggle('is-live', live);
    text.textContent = live ? '' : (everLive ? 'stream offline' : 'connecting…');
  });

  box.addEventListener('click', () => {
    if (!box.classList.contains('is-live')) return;
    const target = canvas ?? el;
    if (document.fullscreenElement) {
      document.exitFullscreen?.();
    } else {
      target.requestFullscreen?.();
    }
  });
</script>

</body>
</html>
```

That's the entire change for this task: one `<script type="module">` block inserted between the lightbox markup and `</body>`.
